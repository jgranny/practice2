09/16/2013 *******************Forms:************** *****inputs***** New input types including: search, email, url, date, tel, number using the email type gives useful related keys without having to switch through key menus when you use a mobile version, such as the @ symbol. url also gives more useability on mobile, such as / and .com keys. date is a control for setting the element's value to a string representing a date. This creates a date picker in the browser, and in the mobile version, that lets you pick dates off a calendar and then inputs them for you tel also gives added useablity with mobile devices. on ios, the input changes to the telephone keyboard number creates an up/down arrow that allows you to increase or decrease the input value by clicking. Mobile brings up the number keys at the top of the keyboard to allow easier number inputs range is an imprecies control for setting a number. creates a slider that will let you change the number month/week - creates input boxes which allow you to select a whole month or a whole week respectively. time/datetime-local - creates an input box that lets you select a time, or a time that lets you select a date AND a time. respectively. color - lets us set a string that represents a color. this creates a toggle that gives us a color picker to select a color. *****form elements***** datalist - this will give autocomplete options that are specified in the options below. The list "browsers" tells the input to link to the datalist with the id "browsers" then it will create autocomplete options for the user as they type *****form attributes***** placeholder, autofocus, autocomplete, required, pattern, list, multiple, novalidate, formnovalidate, form, formaction, formenctype, formmethod, formtarget placeholder - allows you to specify a message that is shown inside the input until the user starts typing, then will return when focus is lost on the blank input autofocus - automatically focuses to the specified input when the page is rendered required - when the form is submitted, the user will be notified of an error if the field is left blank. pattern - accepts a javascript regular expression that can be used to validate the form field to match a pattern. Will notify the user with an error until the pattern is matched. ****************CSS3 Styles*****************
